# Demoblaze QA Testing Project

## Project Overview

This project contains the analysis, design, and execution of manual testing activities performed on the Demoblaze web application.

The main objective was to validate the application's core functionalities, identify defects, and document testing results following QA best practices.

## Application Under Test

🔗 https://www.demoblaze.com

## Project Objectives

- Analyze application functionality
- Design and execute test cases
- Perform manual testing
- Identify and report defects
- Document test evidence and results

## Scope of Testing

The following functionalities were tested:

- User Registration
- User Login
- User Logout
- Product Browsing
- Add Product to Cart
- Remove Product from Cart
- Place Order Process
- Form Validation
- Success and Error Messages

## Testing Types

- Functional Testing
- Positive Testing
- Negative Testing
- Exploratory Testing
- Usability Validation

## Test Cases Summary

A total of 5 test cases were designed and executed to validate the Registration and Login functionalities.

| ID     | Priority | Module  | Test Case                                     |
| ------ | -------- | ------- | --------------------------------------------- |
| TC-001 | High     | Sign Up | User Registration                             |
| TC-002 | High     | Login   | User Login                                    |
| TC-003 | Medium   | Login   | Login After Second Attempt                    |
| TC-004 | Medium   | Login   | Login After Third Attempt                     |
| TC-005 | Medium   | Login   | Account Locked After Multiple Failed Attempts |
